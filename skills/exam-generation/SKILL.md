# 试题设计

## Activation Rules

**触发场景：**
- 用户说「出套题」「考试设计」「试卷」
- 用户需要设计期中/期末考试、单元测验或随堂练习
- 需要试卷结构、题型分布和评分标准

**不触发场景：**
- 用户只问某道题的答案（而非出题）
- 需要的是课后作业（调 homework-grading）

## Agentic Protocol

Step 1: 需求收集 — 课程范围、考试时长、总分、题型偏好、难度要求
Step 2: 蓝图设计 — 知识点分布矩阵 + 题型权重 + 难度梯度
Step 3: 逐题生成 — 按蓝图输出每道题，标注知识点和难度
Step 4: 答案与评分 — 参考答案 + 评分标准 + 采分点
Step 5: 输出 — 完整试卷+答案

## Core Operation Models

| # | 模型 | 核心命题 |
|---|------|---------|
| 1 | **考核一致性** | 试题必须直接对应教学目标 |
| 2 | **难度梯度** | 基础:中等:拔高 ≈ 5:3:2 |
| 3 | **布鲁姆覆盖** | 试题覆盖记忆到创造多层级 |

## Output Style

- 先输出试卷蓝图（知识点×题型矩阵）再逐题展开
- 每道题标注：知识点、能力层级、难度（易/中/难）
- 主观题附详细评分标准和示例答案
- 客观题直接给答案

## Output Modes

| Mode | 触发 | 输出 |
|------|------|------|
| 完整试卷 | 默认 | 蓝图+全部试题+答案+评分标准 |
| 题库补充 | "加几道题" | 针对特定知识点生成补充题 |
| 快速测验 | "小测验" | 5-10 道题，轻量级 |
| 口试/面试题 | "面试"或"口试" | 开放式问题+评价要点 |

## Boundary Rules

1. 不生成超出用户指定课程范围的内容
2. 难度需基于用户描述的学生水平调整
3. 主观题评分标准提供采分点框架，具体分值用户定
4. 不保证试题与特定教材的逐页对应
5. 信息截止 2025 年
