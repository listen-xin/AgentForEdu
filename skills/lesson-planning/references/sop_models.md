# SOP Models — 教案制作完整操作模型卡片

每张卡片 8 字段。运行时协议 Step 1：扫描「When to use」匹配模型。

---

## H1: 双轨目标结构

**When to use**: 任何教案的目标叙写环节。尤其当教师面临"既要应对考试又要培养素养""既要写清楚目标又要保留探究空间"的矛盾时。

**Inputs**: 课题/单元内容、对应课程标准条目、考试大纲相关要求。

**Action**:
1. 从课标/考纲中提取本课的"必须掌握"最低标准项（知识+技能）
2. 将每项转写为 ABCD 格式的行为目标（Actor/Behavior/Condition/Degree），归类为**收敛性轨道**
3. 设计 2-4 个 Essential Questions（开放、可辩论、跨情境），归类为**发散性轨道**
4. 在教案中分两区域展示，标注各自的评估方式（收敛性用标准参照，发散性用表现性评价）
5. 检查：收敛性轨道是否覆盖了所有课标硬性要求？发散性轨道是否真正开放（不同学生可以到达不同深度）？

**Output**: 目标双轨对照表（收敛性 3-5 条 + 发散性 2-4 条 + 评估方式对照）。

**Evidence**: R1-learning_objectives.md — 基于 Eisner 表现性目标理论 + Wiggins Essential Questions + 中国核心素养双重需求。

**Failure Mode**: 发散性轨道写成"有标准答案的引导问题"→ 退回用"为什么…""在多大程度上…""如果…会怎样"句式改写；收敛性轨道使用不可检测动词（"了解""理解""掌握"）→ 替换为 Bloom's 可测动词表对应动词。

**Boundary**: 文科（语文/历史）发散性目标占比应高于理科（数学/物理）。艺术/创作类课程可以发散性为主、收敛性为辅。

**Confidence**: high（双轨结构在多个学科案例中验证有效，但大规模对照实验缺乏）。

## H2: Hinge Question 触发诊断

**When to use**: 备课时为关键概念节点设计诊断问题；在课中检测学生是否真正理解（而非虚假掌握）。

**Inputs**: 本课核心概念、学生常见迷思概念列表。

**Action**:
1. 选择一个本课最核心的前置概念，设计一个多选 Hinge Question
2. 设计原则：(a) 每个错误选项对应一个特定的常见迷思概念；(b) 正确答案不能靠猜测——学生必须真正理解才能选对；(c) 全班响应+教师扫描 <2 分钟
3. 预案：错误率 >25%（基础概念）或 >35%（进阶概念）→ 启动微型补救 + 同伴教学
4. 将 Hinge Question 安排在课堂开始后 10-15 分钟处
5. 在教案中标注：[HQ-1] + 位置 + 阈值 + 超标预案

**Output**: Hinge Question + 选项对应的迷思概念映射 + 超标预案。

**Evidence**: R2-learner_analysis.md — 基于 Black & Wiliam 形成性评价 + Wiliam (2011) Embedded Formative Assessment + Nathan 专家盲点研究。

**Failure Mode**: Hinge Question 设计为"一看就会"的简单题（失去诊断功能）→ 检查：正确答案是否要求真正的概念理解而非识别？错误选项是否对应真实的学生迷思概念？

**Boundary**: 大班额（40+）需要抽样回答（选 8-10 名代表性学生同时举牌），而非逐一检查。对低年级，可用手势信号替代书面选择。

**Confidence**: high（Hinge Question 技术被 Wiliam 等广泛验证，但具体阈值 25%/35% 是启发式的）。

## H3: 支架分级降级协议 (Fade-by-Tier)

**When to use**: 为任何需要脚手架支持的教学环节设计支架时。尤其当教师发现学生"在有支架时全会、撤了支架就不会"时。

**Inputs**: 核心学习任务描述、学生当前独立完成水平。

**Action**:
1. 为每个核心支架设计 3-4 个层级：L3 高支架（完整步骤清单/句子框架）→ L2 中支架（句首词/关键词提示）→ L1 低支架（仅目标提示/检查点）→ L0 无支架（完全独立）
2. 每层设定明确的升级/降级触发条件（如"连续 3 次有支架任务正确率 >85%"触发降级）
3. 在教案中标注每个任务当前使用的支架层级
4. 每节课保留 ≥5 分钟"独立窗口"时间——强制使用 L0 或下一目标层级
5. 降级时在教案中记录：降级原因 + 预期困难 + 观察点

**Output**: 支架层级表（4 层 × 触发条件 + 独立窗口安排）。

**Evidence**: R2-learner_analysis.md + R4-key_difficulty_breakthrough.md — 基于 Van de Pol et al. (2010) 脚手架元分析 + Fisher & Frey 渐进释责修正 + Pearson & Gallagher (1983)。

**Failure Mode**: 只有搭建没有撤除（永久性脚手架）→ 强制检查：教案中是否标注了每个支架的撤除条件？"全班正确率 100%"是红旗信号（支架过强或任务过易）。

**Boundary**: 支架撤除的最佳时机（多少次成功后撤？）缺乏严格的实验证据，当前参数（连续 3 次 >85%）是启发式的。不同技能类型（陈述性 vs 程序性 vs 元认知）可能需要不同参数。

**Confidence**: medium（支架层级设计有理论依据，但撤除参数缺乏大规模验证）。

## H4: BOPPPS 六阶段时间锚定

**When to use**: 设计一节 40-50 分钟的新授课时。尤其适合教师讲授时间过长需要控制"教-学"比例的课型。

**Inputs**: 教学课题、目标、核心活动设想。

**Action**:
1. Bridge-in (3-5min): 激活先验知识 + 建立学习动机（故事/问题/演示/前测结果反馈）
2. Objective (2min): 明示本课学习目标（对内版：教师内部使用；对外版：学生可见的简短版）
3. Pre-assessment (3-5min): 诊断起点（Hinge Question / 快速投票 / KWL 的 K 和 W）
4. Participatory Learning (18-20min): 核心教学活动，教师讲授 ≤10 分钟，其余为学生主动加工时间
5. Post-assessment (5-8min): 检测达标情况（Exit Ticket / Muddiest Point / 快速测验）
6. Summary (3-5min): 串联回顾 + 预告下节课 + 学后反思引导

**Output**: 六阶段时间分配表 + 每阶段教师行为/学生行为/评价嵌入点。

**Evidence**: R3-activity_sequencing.md — 基于 ISW (Instructional Skills Workshop) BOPPPS 框架 + 崔允漷学历案 1008 节课观察数据。

**Failure Mode**: Participatory Learning 变成教师讲授 18 分钟 → 检查：学生主动加工时间是否 ≥10 分钟？是否包含至少 1 个需要学生产出的活动（讨论/练习/操作）？

**Boundary**: 复习课/实验课/项目课不适用标准 BOPPPS 配比。60 分钟+ 长课用 H5 认知呼吸点代替单次完整循环。

**Confidence**: high（BOPPPS 在加拿大和中国高校均有广泛应用，中小学适配需微调时间参数）。

## H5: 认知呼吸点切分法则

**When to use**: 课堂时长为 60-90 分钟时；或内容包含 2 个以上独立子目标时；或 BOPPPS 标准比例已不适用时。

**Inputs**: 长课时总时长、2-4 个子学习目标、核心活动复杂度。

**Action**:
1. 将长课时按"子目标达成"边界切分为 2-4 个迷你循环（每个 15-25 分钟）
2. 每个迷你循环含：迷你 Bridge-in (1min) → 核心活动 → 快速后测 → 微休息
3. 微休息 90 秒：非结构化（站立伸展/自由提问/同伴闲聊），不布置任务
4. 循环间通过"过渡句"衔接（"刚才我们搞清楚了 X，接下来看 Y 和 X 有什么关系"）
5. 最后一个循环预留 5-8 分钟完整 Post-assessment + Summary，覆盖全部子目标

**Output**: 多循环时间轴 + 切分点说明（为什么在此处切分）+ 微休息安排。

**Evidence**: R3-activity_sequencing.md — 基于 Sharpe, Trotter & Hale (2025) 微休息实验 + Sweller 认知负荷理论 + Bradbury (2016) 注意力批判综述。

**Failure Mode**: 机械按 10 分钟定时切块（不管内容逻辑完整性）→ 检查：每个切分点是否落在"一个子目标达成、下一个目标开启"的逻辑转折处？

**Boundary**: 复杂概念建构（如数学定理推导）可能需要连续 30 分钟深度思考——此时不应机械切分。微休息的 90 秒参数来自大学讲座实验，中小学的最佳参数可能不同。

**Confidence**: medium（框架逻辑坚实，但微休息的剂量效应在各学段/学科中缺乏系统验证）。

## H6: 关键度-前置依赖度弹性决策

**When to use**: 备课阶段为每个教学环节标注优先级；课中发现进度落后需要临场决策时。

**Inputs**: 本课所有教学环节清单、单元目标体系、后续单元依赖关系。

**Action**:
1. 备课时逐一标注每个环节：关键度（1-5，对本课/单元目标的贡献度）、前置依赖度（1-5，对后续单元支撑度）
2. 生成决策矩阵：双高（≥4, ≥4）= 不可跳过（必须保证练习+评估）；高关键低前置（≥4, ≤2）= 可精简讲不练；低关键（≤2）= 可课后自学（学历案材料）
3. 进度落后时：优先压缩"低关键"环节 → 其次精简"高关键低前置"练习时间 → 最后才考虑跳过练习（仅限双低环节）
4. 在教案中用颜色标注：🔴 不可跳过 🟡 可精简 🟢 可课后

**Output**: 教学环节优先级矩阵 + 当堂弹性决策预案。

**Evidence**: R3-activity_sequencing.md — 综合 BOPPPS 时间分配 + 认知负荷理论 + 课堂节奏弹性需求。

**Failure Mode**: 把所有环节都标为双高（失去区分度）→ 检查：双高环节是否超过总环节数的 50%？如果超过，重新审视——不是所有内容都同等重要。

**Boundary**: 该矩阵不适用于"所有内容都是前置依赖"的线性强依赖学科（如某些数学单元）。此时应优先压缩导入/复习环节而非核心内容。

**Confidence**: medium（框架源自教学实践智慧，缺乏对照实验验证，但逻辑自洽）。

## H7: 重难点四象限分离诊断

**When to use**: 每次教案中标注"教学重点"和"教学难点"之前——阻止不区分重点/难点就写突破策略的常见错误。

**Inputs**: 本课教学内容、课标要求、学情诊断数据。

**Action**:
1. 列出本课所有教学内容项
2. 逐项判断：(a) 是否是课标规定的最核心内容？（是=重点）(b) 根据前测/经验，学生是否大概率在这里卡住？（是=难点）
3. 将每项归入四象限之一：重合型（重点+难点）、重点非难点、难点非重点、分离型（都不是）
4. 不同象限不同策略：重合型 → H8 认知解剖+多策略并行；重点非难点 → 确保覆盖但不需要深度突破；难点非重点 → H8 非重点难点三问分流；分离型 → 删除或降为背景信息

**Output**: 教学内容-重点/难点四象限判断表 + 每象限策略分配。

**Evidence**: R4-key_difficulty_breakthrough.md — 综合教学重点/难点的定义辨析 + 新手教师常见错误分析。

**Failure Mode**: 把所有内容都标为"重合型"（极度高估难度）或全部"重点非难点"（极度低估难度）→ 交叉验证：用学情前测数据校准难点判断。

**Boundary**: 适用于认知类学科。技能/艺术类学科的"难点"定义需要调整——更多是动作协调/感知精确度而非认知门槛。

**Confidence**: high（重点/难点的区分在教学设计文献中有广泛共识，四象限框架是逻辑直接推演）。

## H8: 难点认知解剖七策略

**When to use**: H7 诊断出"重合型难点"或"难点非重点中的阻塞型困惑"时。

**Inputs**: 已确定的难点、学生基础水平、可用课时。

**Action**:
1. 解剖难点：列出学生跨越这个难点需要依次完成的 3-5 个认知子步骤
2. 识别每个子步骤的障碍类型：记忆型/理解型/技能型/态度型/语言型
3. 从七策略中配对：记忆型 → 先行组织者+口诀化；理解型 → 类比+可视化+问题链；技能型 → 分层递进+示范+练习；态度型 → 情境建构+体验式突破
4. 每个难点至少配 2 种策略（首选+备选），防止单点失效
5. 为每个策略设计支架层级（H3）和撤除计划
6. 核心子障碍用"先战而后教"（逆向渐进释责）：让学生先尝试→认知冲突→策略显性化→教师精讲

**Output**: 难点认知解剖表（子步骤→障碍类型→策略配对→支架层级→撤除条件）。

**Evidence**: R4-key_difficulty_breakthrough.md — 综合七种突破策略类型学 + 五善法 + 脚手架元分析。

**Failure Mode**: 用"多讲几遍"作为唯一策略（混淆信息传递与认知加工）→ 检查：每个子障碍是否配对了至少一种改变认知加工方式的策略（而不是改变呈现次数的策略）？

**Boundary**: 七策略源自实践者总结，缺少效应量数据。不同策略在不同学科/学段的效果差异缺乏系统比较。

**Confidence**: medium（框架综合性强但证据基础为实践智慧而非RCT）。

## H9: 逆向设计三阶段 (Backward Design)

**When to use**: 设计一个完整单元/课时教案时。尤其当教师习惯"先选活动再定目标"的备课顺序时。

**Inputs**: 课程标准条目、教材内容、学情数据。

**Action**:
1. **Stage 1 - 确定预期结果**: (a) 迁移目标（学生将能独立应用什么？）(b) 持久理解/大观念（完整的陈述句，不是主题词）(c) 核心问题/Essential Questions（开放、反复出现、指向大观念）(d) 关键知识与技能
2. **Stage 2 - 确定评估证据**（必须先于 Stage 3 完成！）: (a) 表现性任务（GRASPS 格式：目标/角色/受众/情境/产品/标准）(b) 其他证据（测验/观察/对话/自评）。通过"两个问题检验"：(1) 这个评估能被一个不理解但会套公式的学生蒙混过关吗？(2) 一个真正理解了的学生会因评估形式不当而做不好吗？任一答案为"是"→ 重新设计
3. **Stage 3 - 规划学习活动**: 使用 WHERETO 框架（Where/Hook/Equip/Rethink/Evaluate/Tailor/Organize）组织活动

**Output**: UbD 三阶段教案模板（Stage 1 → Stage 2 → Stage 3 顺序不可逆）。

**Evidence**: R5-assessment_alignment.md + R6-unit_design_reflection.md — 基于 Wiggins & McTighe (2005/2011)。

**Failure Mode**: Stage 2 潦草写"课堂提问+课后作业+单元测验"→ 伪逆向设计。检查：Stage 2 是否包含至少一个需要学生迁移应用的表现性任务？是否有"两个问题检验"通过记录？

**Boundary**: 逆向设计的刚性适用于"可测量学习目标"场景。高度生成性/体验性/创作型教学应降低 Stage 2 的刚性要求。这不是抛弃对齐，而是允许更灵活的证据形式。

**Confidence**: high（UbD 是全球应用最广泛的教学设计框架之一，有大量实施案例但大规模 RCT 有限）。

## H10: 动词对齐检查 (SOLO Alignment)

**When to use**: 教案完成后执行一致性校验时。任何标注为"教-学-评一致性"要求的场景。

**Inputs**: 已完成的教学目标、评价任务、课堂活动。

**Action**:
1. 圈出教学目标中的动词，对应 SOLO 五层级：前结构(无) / 单点(identify, name) / 多点(describe, list) / 关联(explain, analyze, compare) / 抽象扩展(theorize, generalize, reflect)
2. 圈出评价任务中的动词，映射到同层级
3. 圈出课堂活动中的动词，映射到同层级
4. 检查三者是否同层级。目标写"分析"但评价只要"列举"→ 不对齐，需修正
5. 修正方向：将低层级评价/活动提升到目标层级（而非反过来降低目标）

**Output**: 动词一致性检查报告（含不一致项列表和修改建议）。

**Evidence**: R5-assessment_alignment.md — 基于 Biggs 建构一致性 + SOLO 分类学 (Biggs & Tang, 2007)。

**Failure Mode**: 机械追求动词字面一致而忽略认知实质——"分析"在不同学科可能对应不同认知操作。真正的对齐是认知深度的对齐，不是动词字面的匹配。

**Boundary**: SOLO 分类学在人文社科领域的适应性和覆盖度有争议——某些高价值的学习体验（如审美体验）无法被 SOLO 层级清晰定位。

**Confidence**: high（Biggs 建构一致性是高等教育教学设计的核心框架，但中小学的学科适配文献较少）。

## H11: 统摄中心选择矩阵

**When to use**: 设计大单元/主题单元时，需要确定单元的组织方式。

**Inputs**: 单元核心素养目标、学科类型、教师大单元教学经验水平。

**Action**:
1. 判断三个维度：(a) 素养目标偏向：理解型（目标=理解某大观念）→ 大观念；做事型（目标=完成某任务）→ 大任务；探究型（目标=探究某问题）→ 大问题；产出型（目标=做出产品）→ 大项目。(b) 学科适配：语文/历史/政治 → 大观念或大问题；数学/物理 → 大问题或大观念；英语 → 大任务；综合实践 → 大项目。(c) 教师经验：初入者 → 大任务（最贴近日常教学、最易上手）
2. 一个单元只选一种统摄中心。一学期各单元可混合不同类型
3. 按选定类型设计统摄中心的具体表述（大观念必须是完整陈述句，大问题必须开放可辩论，大任务/大项目必须产出具体成果）

**Output**: 统摄中心选择结果 + 理由（三维度分析）+ 统摄中心表述 + 表述质量检查。

**Evidence**: R6-unit_design_reflection.md — 基于崔允漷"单元统摄中心"理论 + Wiggins & McTighe UbD。

**Failure Mode**: 大观念写成主题词（"意象""函数""运动"）→ 退回复核：是否为完整陈述句？是否表达了关系性理解？是否可迁移？

**Boundary**: 当前中国多数学校要求按教材顺序统一进度。大单元"重组"在实践中可能受行政约束——skill 默认假设教师有一定自主权。

**Confidence**: high（崔允漷理论在中国课改中广泛采用，但四种组织方式的效果比较缺乏实证）。

## H12: 反思→Todo→教案修订闭环

**When to use**: 每节课后反思时；发现同一问题反复出现需要深层归因时；学期末汇总理教案迭代记录时。

**Inputs**: 已执行的教案、课堂中学生学习证据（作业/观察/形成性评价数据）。

**Action**:
1. 选用反思模板（建议默认：What Worked / What Didn't / Next Steps，每段 2-3 句上限）
2. 以学习证据为起点：学生在哪些地方表现出理解/困惑？哪些错误模式反复出现？
3. 强制产出至少 1 条可执行 Todo，写入教案对应位置的标注区（如"T1: 增加图示""T2: 调整第3题的问法"）
4. 建立 Todo 追踪：待办 → 已执行（标注执行日期和效果）→ 已放弃（标注原因）
5. 触发双环反思条件：同一问题连续出现 2 次以上 OR 同类错误率 >30% OR 两个班级效果差异显著 → 追问根因（不只是"做什么"而是"为什么会出现这个问题"）

**Output**: What/What/Next 反思记录 + Todo 条目 + 教案修订标注。

**Evidence**: R6-unit_design_reflection.md — 基于 Schon 反思性实践者 + Driscoll/Gibbs 反思框架 + Argyris 双环学习。

**Failure Mode**: 反思空泛（"学生反应良好""教学目标基本达成"）→ 检查：是否引用了具体的学习证据（正确率/典型错误/学生原话）？是否产出了可验证的 Todo？

**Boundary**: 反思频率建议：每节课后只选一节核心课做深度反思（5 分钟），其余课只记一行 Todo。每日全量深度反思会导致反思疲劳和形式化。

**Confidence**: high（反思的理论基础坚实，但"结构化反思→可测量的教学改进"的因果证据链主要为案例研究）。
