## Activation Rules

触发场景：
- 用户说"帮我写一份教案""帮我备课""帮我设计一节课"
- 用户提供教学内容/课题/教材，要求生成教学设计
- 用户问"教学目标怎么写""重难点怎么突破""学情分析怎么做"
- 用户说"帮我做单元整体设计""帮我规划一个单元的教案"
- 用户提及 UbD、逆向设计、学历案、BOPPPS、4A's、教-学-评一体化

不触发场景：
- 用户只问某个学科知识的具体内容（不涉及如何教）
- 用户讨论教育政策/理念但不要求产出教案
- 用户需要课件/PPT 设计（非教案）
- 纯粹的课堂管理/纪律问题
- 学校行政/排课等非教学设计事务

## Agentic Protocol

Step 1: **阶段判定** — 区分用户需求类型：单课时教案 / 单元整体设计 / 某一环节专项指导（如只写教学目标、只做学情分析）

Step 2: **模型匹配** — Read `references/sop_models.md`，扫描「When to use」字段找到匹配的操作模型

Step 3: **分步诊断生成** — 按 6 阶段流程逐阶段执行：
  Phase 1 学情诊断 → Phase 2 目标设计 → Phase 3 评价前置 → Phase 4 活动序列 → Phase 5 重难点突破 → Phase 6 反思嵌入

Step 4: **一致性校验** — 对产出教案执行教-学-评一致性检查（目标动词 = 评价动词 = 活动动词）

Step 5: **输出** — 根据触发场景选择 Output Mode，按对应结构输出

## Core Operation Models

完整操作卡片在 `references/sop_models.md`。摘要：

| # | 模型 | 核心命题 | 来源维度 |
|---|------|---------|---------|
| H1 | **双轨目标结构** | 收敛性目标保底线，发散性目标留探究空间 | R1 教学目标 |
| H2 | **Hinge Question 触发诊断** | 课中 10-15 分钟插诊断题，错误率>30% 立即启动预案 | R2 学情分析 |
| H3 | **支架分级降级协议** | 每个支架配 3-4 层级+撤除条件，独立窗口校准真实 ZPD | R2 学情分析 |
| H4 | **BOPPPS 六阶段时间锚定** | 45 分钟按 B-I-O-P-P-S 结构化分配，讲授≤10 分钟 | R3 活动序列 |
| H5 | **认知呼吸点切分** | 长课时以子目标为单位切分，循环间 90 秒微休息 | R3 活动序列 |
| H6 | **关键度-前置依赖度弹性决策** | 备课时双标注，进度落后按矩阵取舍 | R3 活动序列 |
| H7 | **重难点四象限分离诊断** | 区分重点/难点四种关系，各配不同策略组合 | R4 重难点突破 |
| H8 | **难点认知解剖七策略** | 拆子障碍→识别障碍类型→配对策略→设撤除计划 | R4 重难点突破 |
| H9 | **逆向设计三阶段** | Stage 2 评价证据必须先于 Stage 3 活动设计 | R5 评价设计 |
| H10 | **动词对齐检查** | SOLO 五层级：目标动词 = 评价动词 = 活动动词 | R5 评价设计 |
| H11 | **统摄中心选择矩阵** | 大观念/大任务/大问题/大项目四选一，按三维度决策 | R6 单元设计 |
| H12 | **反思→Todo→教案修订闭环** | 反思产可追踪 Todo，直连教案修改，版本迭代 | R6 反思闭环 |

## Output Style

- 先给教案的整体设计思路（1-2 句），再展开具体内容
- 用结构化模板输出教案，不用大段散文
- 引用教学理论时说"Wiggins & McTighe 在 UbD 中指出…"不说"根据 R5 报告…"
- 每个教学环节标注对应的评价嵌入点和预估时间
- 禁止词：「根据框架分析…」「按照模型卡片…」「让我来系统分析…」
- 教案产出后就停，不问"需要我进一步调整吗"（除非有明显对齐问题）

## Output Modes

| Mode | 触发条件 | 输出结构 |
|------|---------|---------|
| **完整教案** | 用户提供课题+年级+学科 | 课题信息 → 学情诊断 → 教学目标(双轨) → 评价证据包 → 教学过程(BOPPPS/4A's) → 重难点突破策略 → 板书设计 → 课后反思模板 |
| **单元设计** | 用户提到"单元""大单元""整章" | 统摄中心 → UbD Stage 1 → 单元目标矩阵 → 课时规划+进阶路线 → 单元评价方案 → 各课时概要 |
| **目标专修** | 用户只问教学目标怎么写 | 学科适配 → 双轨结构 → ABCD 公式 → Bloom's 动词审校 → Essential Questions 配对 → 正反例对比 |
| **评价设计** | 用户问"怎么评价""怎么检测" | 逆向设计 Stage 2 → 评价证据包 → 动词对齐检查 → 形成性评价技术矩阵 → Sadler 三步闭环 |
| **重难点突破** | 用户问"重难点怎么突破" | 四象限诊断 → 认知解剖 → 策略配对 → 支架层级设计 → 撤除计划 → 典型错误预判 |
| **反思改进** | 用户提供已执行的教案要求改进 | What Worked / What Didn't / Next Steps → Todo 生成 → 教案修订标注 |
| **学历案** | 用户要求学历案格式 | 预学任务 → 研学活动(学生立场) → 固学拓展 → 学后反思支架(三阶六级) |

## Boundary Rules

1. **学科边界**：本 skill 覆盖认知类学科（语数外理化生政史地）。技能/艺术/体育类需调整框架刚性，允许更大的活动灵活性
2. **学段边界**：主要适用于中小学。大学/成人培训需调整课时结构和评价粒度
3. **不替代教师判断**：生成的教案是结构化草稿，教师需根据实际学情和课堂生成调整。AI 不了解你的具体学生
4. **框架适用边界**：逆向设计适用于"可测量学习目标"场景。高度生成性/体验性教学（自由创作、开放讨论）应降低对齐刚性
5. **不提供学科内容**：本 skill 提供的是教学设计方法，不替代学科专业知识。具体知识点讲解仍需学科教师把关
6. **班级规模假设**：默认 20-35 人班级。40+ 人大班需将诊断方法抽样化，支架协议需简化
7. **信息截止**：教学框架和课标要求基于 2025 年公开信息。地方性教研要求以当地文件为准
8. **深度: deep · 质量: standard**
