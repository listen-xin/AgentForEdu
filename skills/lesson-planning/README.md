# 教案制作 · Lesson Planning

> 「教完了不等于学会了——教案是让这两者对齐的蓝图。」

将 UbD、BOPPPS、学历案、UDL 等主流教学设计框架融合为一个可执行的操作系统。从学情诊断到课后反思，全流程引导教师产出教-学-评一致的高质量教案。

## 安装

```bash
cp -r output/lesson-planning-skill ~/.claude/skills/lesson-planning/
```

## 触发方式

- 说"帮我写一份教案""帮我备课""帮我设计一节XX课"
- 提供教学内容要求生成教学设计
- 问"教学目标怎么写""重难点怎么突破""学情分析怎么做"
- 说"帮我做单元整体设计""帮我规划单元教案"
- 提及 UbD、逆向设计、学历案、BOPPPS、教-学-评一体化

## 核心操作模型

| # | 模型 | 一句话 |
|---|------|--------|
| H1 | 双轨目标结构 | 收敛性目标保底线，发散性目标留探究空间 |
| H2 | Hinge Question 触发诊断 | 课中 10-15 分钟插诊断题，错误率超标即启动预案 |
| H3 | 支架分级降级协议 | 每个支架配 3-4 层级+撤除条件，独立窗口校准真实水平 |
| H4 | BOPPPS 六阶段时间锚定 | 45 分钟结构化分配，讲授≤10 分钟 |
| H5 | 认知呼吸点切分 | 长课时以子目标切分，循环间 90 秒微休息 |
| H6 | 弹性时间决策 | 备课时双标注，进度落后按矩阵取舍 |
| H7 | 重难点四象限分离 | 区分重点/难点四种关系，各配不同策略 |
| H8 | 难点认知解剖七策略 | 拆子障碍→识别类型→配对策略→设撤除计划 |
| H9 | 逆向设计三阶段 | Stage 2 评价前置，两问题检验防伪逆向 |
| H10 | 动词对齐检查 | SOLO 五层级确保目标=评价=活动 |
| H11 | 统摄中心选择矩阵 | 大观念/大任务/大问题/大项目四选一 |
| H12 | 反思→Todo→教案修订 | 反思产可追踪行动，直连教案迭代 |
